# Автоматизация тестирования мобильного приложения FMHandroid

Данный репозиторий содержит проект автоматизации UI-тестов для мобильного приложения **FMHandroid** («Мобильный хоспис»). В рамках работы успешно автоматизировано и стабилизировано **85 тест-кейсов**, покрывающих критический функционал (авторизация, навигация, фильтрация, создание и редактирование новостей).

---

## 1. Как скачать и запустить проект локально

### Шаг 1. Клонирование репозитория
Склонируйте проект к себе на компьютер через терминал:
```bash
git clone <https://github.com/NataliaSerdakova/Graduation_Project.git>
```

### Шаг 2. Импорт в IDE
1. Откройте **Android Studio**.
2. Выберите пункт **File ➔ Open** (или *Open an Existing Project*).
3. Укажите путь к корневой папке склонированного проекта и дождитесь завершения синхронизации Gradle.

### Шаг 3. Настройка окружения
Для стабильного и успешного прогона тестов запустите эмулятор (AVD) со следующими характеристиками:
* **Версия Android (API):** API 29 (Android 10.0).

---

## 2. Тестовые учетные данные (Данные для входа)

Все тесты используют единый валидный аккаунт. Согласно правилам централизованного управления конфигурацией, данные вынесены в статические методы вспомогательного класса `DataHelper`:

* **Логин:** `login2` *(метод DataHelper.getValidLogin())*
* **Пароль:** `password2` *(метод DataHelper.getValidPassword())*

*В методах `@Before` каждого класса настроена автоматическая проверка сессии (if-else) и сброс состояния через `logout()`, что гарантирует полную независимость тестов при запуске всей цепочкой.*

---

## 3. Инструкция по запуску автотестов

Вы можете запускать тесты любым удобным способом:

### Способ 1. Через графический интерфейс Android Studio 
1. Откройте необходимый тестовый класс (например, `EditNewsTest`).
2. Для запуска **всех тестов класса одновременно** нажмите на **зелёную стрелочку** слева от названия самого класса (строка `public class ...`).
3. Для запуска отдельного сценария нажмите на зелёную стрелочку напротив конкретного метода `@Test`.

### Способ 2. Прогон через Терминал
Чтобы запустить прогон всех существующих тестовых классов одной командой через консоль, в нижней панели Android Studio откройте вкладку **Terminal** и введите:
* **Для Windows (PowerShell):** `.\gradlew connectedAndroidTest`
* **Для Mac / Linux:** `./gradlew connectedAndroidTest`

## 4. Работа с отчетами Allure

1. **Готовый архив результатов:** Результаты последнего успешного прогона упакованы в корень проекта под именем **`allure-results.zip`**. Подробный разбор 8 найденных дефектов, логических особенностей и метрик зафиксирован в файле **`Result.md`**.
2. **Локальная генерация отчета:** Чтобы самостоятельно развернуть интерактивный веб-отчет в браузере из папки результатов, введите в Терминале команду:
   ```bash
   allure serve allure-results
   ```
